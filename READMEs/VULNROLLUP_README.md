# `subp_vuln_rollup.py`

Roll up Black Duck vulnerabilities from child/subproject project versions into a parent project/version context.

## Auth

Use either environment variables or CLI flags.

### Environment variables

```textmate
export BLACKDUCK_URL="https://blackduck.example.com"
export BLACKDUCK_API_TOKEN="..."
```


### Equivalent flags

```textmate
--bd-url https://blackduck.example.com
--api-token ...
```


## Common run

```textmate
python subp_vuln_rollup.py \
  --parents-csv parent_projects.csv \
  --threshold 7 \
  --out findings.csv \
  --insecure \
  --timeout 30 \
  --retries 1 \
  --page-limit 500 \
  --failures-out failed-rollup-relationships.csv
```


## Modes

### CSV mode

Uses parent/child relationships from a CSV generated by parent discovery tooling.

```textmate
--parents-csv parent_projects.csv
```


Optional filters:

```textmate
--parent-project "Parent Name"
--parent-version "1.0"
```


### Single parent discovery mode

Discovers subprojects from one parent project/version BOM.

```textmate
--parent-project "Parent Name"
--parent-version "1.0"
--depth 2
```


Optional BOM-name fallback:

```textmate
--resolve-bom-names
```


## Vulnerability filtering

```textmate
--threshold 7
--score-field overallScore
```


| Flag | Description | Default |
|---|---|---|
| `--threshold` | Minimum numeric score included in output. | Required/selected by user |
| `--score-field` | Vulnerability score field to evaluate. | `overallScore` |

## Targeting / retry filters

Use these flags to rerun one failed relationship.

```textmate
--only-child-href "https://.../api/projects/.../versions/..."
--only-child-project "Child Project"
--only-child-version "Child Version"
```


`--only-child-href` is the most precise retry selector.

## Output

```textmate
--out findings.csv
--json
--failures-out failed-rollup-relationships.csv
```


| Flag | Description |
|---|---|
| `--out findings.csv` | Writes findings to the specified CSV file. |
| `--out -` | Writes findings to stdout. |
| `--json` | Writes findings as JSON instead of CSV. |
| `--failures-out failed-rollup-relationships.csv` | Writes failed child relationship attempts. |

### Main CSV fields

```
parent_project,parent_version,parent_version_href,subproject_path,subproject,
subproject_version,subproject_version_href,relationship_detection_method,
component,component_version,vulnerability,score_field,score,severity,
blackduck_url,rollup_key
```


## HTTP / TLS

```textmate
--timeout 30
--retries 1
--retry-delay 2.0
--page-limit 500
--insecure
```


| Flag | Description |
|---|---|
| `--timeout` | Per-request timeout in seconds. |
| `--retries` | Retry count for transient/network failures. |
| `--retry-delay` | Base retry delay in seconds. |
| `--page-limit` | Black Duck paginated API page size. |
| `--insecure` | Disables TLS verification. |

## API cache

Persistent API cache is enabled by default.

```textmate
--api-cache subp_vuln_rollup_cache.json
--api-cache-max-age-hours 20
--api-cache-max-entries 5000
```


Disable persistent cache:

```textmate
--no-api-cache
```


Force fresh API reads and rebuild the cache:

```textmate
--refresh-api-cache
```


### Recommended daily cron behavior

- Default `--api-cache-max-age-hours 20` avoids using yesterday’s vulnerability data on a daily run.
- If a run fails and is rerun shortly after, the same cache can be reused.
- Use `--refresh-api-cache` for guaranteed fresh nightly output.

## Debug

```textmate
--debug
```


Prints API/cache progress to stderr.

Useful interactively; usually omit in cron unless logs are needed.

## Cron example

```textmate
python /opt/blackduck/subp_vuln_rollup.py \
  --parents-csv /opt/blackduck/parent_projects.csv \
  --threshold 7 \
  --out /opt/blackduck/findings.csv \
  --timeout 30 \
  --retries 1 \
  --page-limit 500 \
  --api-cache /opt/blackduck/subp_vuln_rollup_cache.json \
  --api-cache-max-age-hours 20 \
  --failures-out /opt/blackduck/failed-rollup-relationships.csv
```
